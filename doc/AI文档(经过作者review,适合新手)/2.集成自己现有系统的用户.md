# 如何在 AntFlow 中接入你自己系统的用户 🐜

> 每个企业都有自己的用户组织架构系统，AntFlow 不强制你用一套通用的用户表，相反，它让你用自己的！

---

## 一、为什么 AntFlow 要这么设计？

用过 Activiti 的老用户都知道：Activiti 内置了一套用户表，但这套表往往和你企业现有的用户系统对不上，改起来非常头疼。

AntFlow 的做法是：**完全接管 Activiti 的用户系统，但用一套极其简单、高度抽象的接口让你对接自己的用户系统**。

你不用改 Activiti 源码，不用迁用户数据，只需要实现几个接口，AntFlow 就能用你自己的用户！

---

## 二、核心概念

### 2.1 两个关键类

AntFlow 只要求你返回两种数据结构：

#### 1. `BaseIdTranStruVo` - 基础用户信息

只有两个字段：

```java
@Data
public class BaseIdTranStruVo {
    private String id;   // 用户 ID
    private String name; // 用户姓名
}
```

**这就够了！** 流程引擎只需要知道“谁在审批”，不需要知道用户的密码、性别、年龄这些信息。

#### 2. `DetailedUser` - 详细用户信息（可选）

只有在需要发通知、邮件时才用到：

```java
@Data
public class DetailedUser {
    private String id;       // 用户 ID
    private String email;    // 邮箱（发邮件通知用）
    private String mobile;   // 手机号（发短信通知用）
    // ... 其他字段都是可选的
}
```

### 2.2 一个核心接口：`AfUserService`

你只需要实现这个接口，AntFlow 就会用你的用户系统！

```java
public interface AfUserService {
    // ========== 必须实现的 4 个方法（核心）==========
  
    // 1. 按名字模糊搜索用户（选人框用）
    List<BaseIdTranStruVo> queryByNameFuzzy(String userName);
  
    // 2. 按用户 ID 批量查用户
    List<BaseIdTranStruVo> queryUserByIds(Collection<String> userIds);
  
    // 3. 按单个用户 ID 查用户
    BaseIdTranStruVo getById(String id);
  
    // 4. 按公司名模糊搜索（多公司场景用，没有就返回空）
    List<BaseIdTranStruVo> queryCompanyByNameFuzzy(String companyName);
  
    // ========== 高级功能（用到再实现）==========
  
    // 查直属领导、查部门领导、查 HRBP 等...
    // 这些方法等你用到对应的审批规则时再实现就行！
}
```

## 三、三种接入方式（选一种就行）

### 方式一：直接修改 `UserServiceImpl`（最简单，快速上手）

AntFlow 自带了一个 `UserServiceImpl`，里面用的是 Demo 表 `t_user`。你只需要把里面的 SQL 改成你自己的！

**步骤：**

1. 打开文件：`antflow-base/src/main/java/org/openoa/base/service/UserServiceImpl.java`
2. 把里面的 `UserMapper` 调用换成你自己的 DAO/Service

**示例：**

假设你自己的用户 Service 叫 `MyUserService`，有个 `searchUser()` 方法：

```java
@Service("afUserService")
public class UserServiceImpl implements AfUserService {
  
    // 注入你自己的用户 Service
    @Autowired
    MyUserService myUserService;
  
    @Override
    public List<BaseIdTranStruVo> queryByNameFuzzy(String userName) {
        // 1. 从你自己的系统查用户
        List<MyUser> myUsers = myUserService.searchUser(userName);
      
        // 2. 转成 AntFlow 需要的格式
        return myUsers.stream()
            .map(user -> BaseIdTranStruVo.builder()
                .id(user.getUserId())    // 你的用户 ID
                .name(user.getRealName()) // 你的用户姓名
                .build())
            .collect(Collectors.toList());
    }
  
    @Override
    public List<BaseIdTranStruVo> queryUserByIds(Collection<String> userIds) {
        // 同样的方式，从你自己的系统查，然后转成 BaseIdTranStruVo
        List<MyUser> myUsers = myUserService.listUsersByIds(userIds);
        return convertToVoList(myUsers);
    }
  
    @Override
    public BaseIdTranStruVo getById(String id) {
        MyUser user = myUserService.getUserById(id);
        return convertToVo(user);
    }
   
  
    // 其他方法先不用管，等用到再实现！
}
```

就这么简单！核心就是：**从你那里查数据，转成 id + name 的格式**。

---

### 方式二：自己实现 `AfUserService` + `@Primary`（推荐，更优雅）

如果你不想修改 AntFlow 自带的代码，可以自己写一个实现类，用 `@Primary` 覆盖默认的。

**步骤：**

1. 创建你自己的实现类：

```java
@Service
@Primary  // 加上这个注解，AntFlow 就会用你的实现！
public class MyAfUserServiceImpl implements AfUserService {
  
    @Autowired
    MyUserService myUserService;
  
    // 实现上面说的 4 个核心方法...
}
```

2. 搞定！AntFlow 会优先用你的实现。

---

### 方式三：不查数据库，从接口获取（灵活！）

用户信息不一定存在数据库里，也可以是调用第三方接口！AntFlow 不关心你从哪拿数据，只关心返回格式。

**示例：**

```java
@Service
@Primary
public class MyAfUserServiceImpl implements AfUserService {
  
    @Autowired
    MyUserApiClient myUserApiClient; // 你的第三方用户 API 客户端
  
    @Override
    public List<BaseIdTranStruVo> queryByNameFuzzy(String userName) {
        // 调用第三方接口
        ApiResponse<List<MyUserDTO>> response = myUserApiClient.searchUser(userName);
      
        // 转成 AntFlow 需要的格式
        return response.getData().stream()
            .map(dto -> BaseIdTranStruVo.builder()
                .id(dto.getUserId())
                .name(dto.getName())
                .build())
            .collect(Collectors.toList());
    }
  
    // ... 其他方法同理
}
```

## 四、高级功能什么时候实现？

`AfUserService` 里还有很多方法，比如：

- `queryLeadersByEmployeeIdAndTier()` - 查 N 级领导
- `queryEmployeeHrpbByEmployeeIds()` - 查 HRBP
- `queryDepartmentLeaderByIds()` - 查部门领导
- `getEmployeeDetailById()` - 查详细信息（发通知用）

**不用一开始就全实现！**

- 先实现 4 个核心方法，流程就能跑起来了
- 等你用到“直属领导审批”这个规则时，运行会报错，提示你 `queryEmployeeDirectLeaderByIds()` 没实现
- 这时候再去实现这个方法就行！

## 五、常见问题

### Q: 我的用户表 ID 是 Long 类型，不是 String 怎么办？

A: 转成 String 就行！`String.valueOf(userId)`

### Q: 我的用户表字段不叫 `id` 和 `name`，叫 `user_id` 和 `real_name` 怎么办？

A: 转的时候映射一下就行：

```java
BaseIdTranStruVo.builder()
    .id(user.getUserId())    // 你的字段名
    .name(user.getRealName()) // 你的字段名
    .build()
```

### Q: 我想在用户选择器里显示更多信息（比如部门），可以吗？

A: 目前 `BaseIdTranStruVo` 只有 id 和 name，但 name 字段你可以拼上其他信息：

```java
BaseIdTranStruVo.builder()
    .id(user.getUserId())
    .name(user.getName() + " - " + user.getDeptName()) // 拼上部门
    .build()
```

## 六、总结

AntFlow 接入用户系统的核心就是：

1. 📦 **封装**：不管你的用户系统是什么样，封装成 `id + name`
2. 🔌 **实现**：实现 `AfUserService` 的 4 个核心方法
3. 🎯 **覆盖**：用你的实现替换默认实现

**就这么简单！** 不用改 Activiti，不用迁数据，1 小时搞定对接！
