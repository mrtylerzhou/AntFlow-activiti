## 一、什么是 AntFlow？

AntFlow 是一款基于 Activiti 的**企业级低代码工作流引擎平台**。

### 核心特性
- 🥇 **全网首创虚拟节点(VNode)模式** - 流程流转业务和引擎 API 完全分离
- 😄 **超级简单** - 后端只需要实现一个接口，低代码流程拖拽即可
- 👨‍👨‍👦‍👦 **完全接管用户系统** - 忘掉 Activiti 的用户表，用你自己的
- 🚩 **中国式办公全支持** - 串行、并行、会签、或签、加批、委托、转办、退回任意节点

## 三、10 分钟快速启动

### 第 1 步：数据库配置

1. 新建数据库，取名 `antflow`
2. 修改配置文件：`antflow-web/src/main/resources/application-dev.properties`
```properties
# 数据库连接
spring.datasource.url=jdbc:mysql://localhost:3306/antflow?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
spring.datasource.username=root
spring.datasource.password=你的密码
```

### 第 2 步：初始化数据库

执行 `script/` 目录下的 SQL：
```bash
# 1. Activiti 引擎表
act_init_db.sql

# 2. AntFlow 业务表
bpm_init_db.sql

# 3. Demo 数据（生产环境可选）
bpm_init_db_data.sql
```

### 第 3 步：启动后端

```bash
# 进入项目根目录
cd Antflow

# 用 Maven 启动
./mvnw clean install -DskipTests
cd antflow-web
../mvnw spring-boot:run
```

或者直接在 IDE 里运行 `antflow-web` 模块的主类。

### 第 4 步：启动前端

```bash
# 进入前端目录
cd antflow-vue

# 安装依赖
npm install --registry=https://registry.npmmirror.com

# 启动
npm run dev
```

访问：**http://localhost:80**

## 五、核心概念：虚拟节点(VNode)

虚拟节点是 AntFlow 的灵魂，一句话解释：
> **虚拟节点 = 流程流转业务（你写的代码） + Activiti/Flowable 引擎 API（AntFlow 封装好的）**

### 为什么要用虚拟节点？
- 🔓 **和引擎解耦**：以后想从 Activiti 迁到 Flowable？不用改业务代码
- 🧠 **零流程引擎知识**：不用学 Activiti 的 200+ 个 API，AntFlow 都帮你封装好了
- 🎛️ **更灵活可控**：加批、转办、退回任意节点，想怎么改就怎么改
